## generics-dynamics-statics.md (169 lines)
Polymorphism strategy guide contrasting static dispatch (generics/impl Trait) with dynamic dispatch (dyn Trait). Static dispatch: zero runtime cost via compile-time monomorphization, best for tight loops and known types. Dynamic dispatch: runtime vtable indirection, best for heterogeneous collections and plugin architectures. Includes trade-off table (performance, compile time, binary size, flexibility), object-safety constraints, and ergonomics best practices. Key rules: prefer `&dyn Trait` over `Box<dyn Trait>` for references, use `Arc<dyn Trait + Send + Sync>` for threads, box at API boundary only.
